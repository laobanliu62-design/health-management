# HEARTBEAT.md - 清理规则

## 上下文清理策略（优先级高）

**⚠️ 绝对保留的内容：**
- 项目相关的项目信息（项目管理、进度、计划）
- 重要决策和配置
- 项目任务清单和状态
- 用户明确要求记住的信息
- 记忆文件（memory/*.md）中的关键项目

**可以清理的内容：**
- 闲聊对话
- 重复的检查记录
- 已经解决且存档的小问题
- 临时数据
- **旧日志文件（logs/*.log 超过 3 天）**
- **工具输出结果（已归档到 memory/的）**

**清理频率：** 每 30 分钟自动检查一次

**阈值设置：**
- 80% → 预警（提示用户消息变长）
- 95% → 清理（只清理可丢弃的内容，保留关键项目）
- **100k tokens（模型上限 131k 的 76%）→ 立即预警**

**关键原则：**
1. 宁可多预警，不清重要信息
2. 清理前先备份关键项目到 memory/
3. 如果不知道哪些重要，先问用户
4. **每日自动清理旧日志文件（>3 天）**
5. **监控 token 使用，避免接近模型上限**
